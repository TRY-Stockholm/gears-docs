## What this guide covers

How to mount `@made-people/grease-product` so the shared `Product` data model drives PDPs, product cards, size/variant pickers, and attribute-driven UI. It also shows the Centra-attributes → grease-attributes mapping layer that is always client-specific.

## Prerequisites

- A commerce backend that can produce products — in the monorepo this is a Centra BFF response (`CentraProductPage.productData`) piped through `@made-people/grease-centra`.
- The providers from the [Anatomy of a Gears client](./anatomy-of-a-gears-client.mdx) guide already mounted (`GearsConfigurationProvider`, `TranslationsProvider`, `LinkProvider`, `ImageProcessorProvider`, plus `CentraBootstrap` or your equivalent).
- A per-client attribute-mapper layer. Every client re-interprets Centra's raw attribute keys, so this code lives in `client-frontends/<client>/app/project/mappers/` rather than in the grease package.

## Install

```bash
bun add @made-people/grease-product
```

Pin peer deps at the ranges in the package's `peerDependencies` — currently `react@18.2.0`, `react-dom@18.2.0`, `zod@^3.22.4`, `@made-people/grease-event-emitter@~0.4.1`, `@made-people/react-prop-guard@~1.6.1`, `@made-people/react-use-effect-once@~0.3.1`. The schemas are enforced at runtime with `withZod`, so a mismatched peer will throw on mount.

## Product types

`Product` is a zod discriminated union on `dataModel`. All variants share `ProductShared` (`base: ProductBase` + `siblingProducts`).

```ts
// from @made-people/grease-product
type Product =
  | ProductSimple            // dataModel: "simple"
  | ProductWithVariants      // dataModel: "with_variants"
  | ProductBundle;           // dataModel: "bundle"

type ProductBase = {
  productId: string;
  productKey: string;          // a.k.a. model ID
  slug: string;
  discontinued: boolean;
  content: ProductContent;     // localizable fields
  attributes: Attribute[];     // free-form, client-interpreted
  categories: ProductCategory[];
};

type Attribute<T = unknown> = {
  attributeKey: string;
  attributeLabel: string;
  value: T;
  valueLabel: string;
};
```

`ProductBase`, `content`, and `categories` are backend-agnostic. `attributes` is the escape hatch for anything the base schema doesn't cover (color swatches, fabric, shipping copy, tracking IDs, etc.).

## Attribute mappers

Centra exposes arbitrary attribute keys (`swatch`, `fabric`, `pr_designer`…) with different shapes per tenant. Each client defines a small, pure function per attribute that safeParses the raw value and returns a normalized `Attribute` — or `undefined` if it doesn't match. Real example from Dedicated:

```ts
// client-frontends/dedicated/app/project/mappers/product-attributes/get-color-swatch-attribute.ts
import { z } from "zod";
import { ProductBase, Attribute } from "@made-people/grease-product";

export const COLOR_SWATCH_KEY = "color-swatch";

export function getColorSwatchAttribute(
  productAttributes: ProductBase["attributes"],
): Attribute | undefined {
  const swatchAttribute = productAttributes.find(
    (a) => a.attributeKey === "swatch",
  )?.value;
  const result = z.object({ hex: z.string(), desc: z.string() })
    .safeParse(swatchAttribute);
  if (!result.success) return undefined;
  return {
    attributeKey: COLOR_SWATCH_KEY,
    attributeLabel: COLOR_SWATCH_KEY,
    valueLabel: result.data.desc,
    value: result.data.hex,
  };
}
```

All getters are composed once per product in `get-product-attributes.ts`, then plugged into the Centra mapper as `centraCheckoutApiProductToGreaseProductExtraAttributes`. Downstream code reads them with the zod-guarded hook:

```ts
import { useProductAttributeGetter, useProductBase } from "@made-people/grease-product";
const attr = useProductAttributeGetter(useProductBase())("color-swatch", z.string());
```

## Wire up the provider

There is no bootstrap-level provider for products — `ProductProvider` is mounted per product, on every route or card that renders one. It internally dispatches on `dataModel` to one of three inner providers (simple / with-variants / bundle), fires the `productViewed` event, and exposes `useProductBase`, `useProductVariant`, `useProductPurchaseInfo`, siblings, related products, etc.

```tsx
<ProductProvider
  product={product}
  relatedProducts={relatedProducts}
  context={{ type: "pdp" }}   // or { type: "plp", listId, listName, position }
>
  {children}
</ProductProvider>
```

The `context` prop tags outgoing events so GTM / tracking listeners can distinguish PDP views from impressions in a list.

## PDP (product detail page)

Dedicated's PDP lives in `client-frontends/dedicated/app/gears/routes/centra-product-page-route.tsx`. It maps the BFF payload into a grease `Product` and hands it to the provider:

```tsx
import { ProductProvider } from "@made-people/grease-product";
import { ProductTemplate } from "@made-people/blueprint-product";
import { frontendProductToGreaseProduct } from "../../project/mappers/frontend-product-to-grease-product";

export const CentraProductPageRoute = ({ page, frontendContext }) => (
  <ProductProvider
    product={frontendProductToGreaseProduct(page.productData, frontendContext)}
    context={{ type: "pdp" }}
  >
    <BreadCrumbsProvider breadCrumbs={page.breadCrumbs}>
      <ProductTemplate template="graz" /* … size picker, swatches, … */ />
    </BreadCrumbsProvider>
  </ProductProvider>
);
```

`ProductTemplate` and its children resolve the selected `ProductVariant` via `useProductVariant` / `useSelectedVariantId`; the size picker reads `useProductVariants()` + `useProductVariantDimensions()`; swatches read siblings via `useProductSiblings()`.

Stronger's equivalent route adds a bundle branch that waits for `useCentraSelectionResponse()` and `useCentraBundleResponse()` before building the product with `makeStrongerProductMapper(frontendContext)`.

## PLP (product list / category)

Neither Dedicated nor Stronger has a dedicated category route — category pages render through `storyblok-page-route.tsx`, and search renders through `search-page-route.tsx` with `DepictSearchResultsProvider`. The product-card layer wraps each card in its own provider with `plp` context:

```tsx
// src/frontend/blueprint/search/src/components/product-list/ProductList.tsx
<ProductProvider
  product={product}
  context={{ type: "plp", listId, listName, position }}
>
  <ProductCard />
</ProductProvider>
```

Filter / query composition lives in the listing provider (Depict for these two clients) — grease-product only renders the resulting products.

## Gotchas

- **Attribute getters must be pure** and `undefined`-tolerant. They run during mapping (SSR) and must not touch React or browser APIs.
- **Centra raw shape leaks through.** `ProductBase.attributes[i].value` is `unknown`; always `safeParse` before use. A missing key returns `undefined`, not `null`.
- **`withZod` guards props at runtime.** Passing a mis-shaped `product` to `<ProductProvider>` throws in development — check your mapper output, not the provider.
- **SEO / structured data** is produced separately via `@made-people/grease-structured-data` using `greaseProductToUnformattedStructuredDataProduct`; wire it at bootstrap, not inside `ProductProvider`.
- **Sibling products** are deduped against `product.base.productId` and JSON-cloned to keep React identities stable — don't pass the same object reference twice.

## Related packages

- `@made-people/grease-centra` — ships the `makeCentraCheckoutApiProductToGreaseProductMapper` that most clients wrap.
- `@made-people/grease-depict` / `@made-people/grease-algolia` — the listing/search provider whose results feed the `plp` ProductProvider.
- `@made-people/grease-cart` — reads `itemId` / `ProductPurchaseInfo` from the provider for add-to-cart.
- `@made-people/grease-voyado-engage` — consumes the `voyadoEngageTracking` attribute you attach in the mapper.

## Troubleshooting

- **"Unknown product data model" warning and blank render** — your mapper emitted a `dataModel` string the provider doesn't know. Fix the Centra mapper; don't edit the product schema.
- **`useProductBase must be used within a ProductProvider`** — a card or hook rendered outside the provider (common in Storyblok components that forget to wrap items). Use `useMaybeProductBase` if the caller is legitimately optional.
- **Attribute hooks return `undefined`** — either the Centra side didn't emit the key, or the zod guard rejected it. Log the raw `ProductBase.attributes` and tighten/loosen the schema in the getter.
