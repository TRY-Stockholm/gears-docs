## What you'll learn

How to mount `@made-people/grease-centra` so Centra drives cart, product search, and account flows in a client frontend.

## The four things to know

1. **One provider owns the session.** `CentraProvider` hydrates the Centra selection on mount and exposes it via `useCentraContext()`.
2. **You never call Centra directly.** Consume `grease-cart`, `grease-account`, and `grease-search` hooks — drivers are wired internally.
3. **The session token is the cart.** `CentraLocalStorageTokenHandler` holds it in `localStorage` under `centra-api-token`; its lifetime is the cart's lifetime.
4. **`initializationType` gates the on-mount fetch.** Use `"dontInitialize"` on the checkout finalize page so it doesn't clobber the finalized order.

## Prerequisites

- A React client frontend that can import `@made-people/grease-centra`.
- A Centra Checkout API base URL from your BFF, e.g. `settingsAndContentResponse.centraCheckoutApiBaseUrl`.
- A `FrontendContextV1` that supplies the `CentraQueryContext` fields (`market`, `pricelist`, `language`, optional `site`) plus the mapper-options context (`currency`, `country`, `baseUrl`, `siteSlug`).

## 1. Build the Centra client

```tsx
import {
  createCentraCheckoutAPIClient,
  CentraLocalStorageTokenHandler,
} from "@made-people/centra-checkout-api";

const centraTokenHandler = new CentraLocalStorageTokenHandler(
  30 * 24 * 60 * 60,
);

const centraCheckoutAPIClient = createCentraCheckoutAPIClient({
  baseUrl: settingsAndContentResponse.centraCheckoutApiBaseUrl,
  centraTokenHandler,
});
```

Default TTL on the token handler is one week (`3600*24*7` seconds). The example above overrides it to 30 days (`30 * 24 * 60 * 60` seconds) — pick a value that matches your abandoned-cart window.

## 2. Mount the provider

```tsx
import {
  CentraProvider,
  CentraProviderProps,
  CentraSearchProvider,
} from "@made-people/grease-centra";

const centraProviderProps: CentraProviderProps = {
  checkoutAPIClient: centraCheckoutAPIClient,
  initializationType:
    pageType === "checkoutFinalize" ? "dontInitialize" : "onLoad",
  queryContext: {
    market: frontendContext.market,
    pricelist: frontendContext.pricelist,
    language: frontendContext.language,
    site: { baseUrl, slug: siteSlug },
  },
  mapperOptions: { mappers: strongerCentraMappers, options: { /* … */ } },
};
```

`CentraProvider` renders `CentraGreaseCartProvider` internally, so `useCartState` and friends work anywhere below it. `mapperOptions.mappers` is a partial (defaults fill gaps), but `mapperOptions.options` is required.

> **Skip `"onInteraction"`.** It's declared but not implemented — the provider logs `"CentraProvider: 'onInteraction' not implemented yet"` and does nothing. Use `"onLoad"` or `"dontInitialize"`.

## 3. The render tree

```
CentraProvider                        ← session + query context
  └─ CentraGreaseCartProvider         ← auto-mounted, drives grease-cart
      └─ children                     ← consumers add CentraSearchProvider etc. as siblings
```

`CentraProvider` only mounts `CentraGreaseCartProvider` internally. `CentraSearchProvider` and `AccountProvider` (if used) are added by consumers as siblings inside `children`, not built in.

## 4. Cart, search, account

**Cart.** Consume `grease-cart`. Add-to-cart, voucher add/remove, quantity updates, and `fetchCart` all route through `CentraCartClientDriver` in `src/cart/centra-cart-client-driver.ts`.

```tsx
import { useCartState } from "@made-people/grease-cart";
const cart = useCartState();
```

**Search.** `CentraSearchProvider` has no default — supply `searchFunctionGetter`. Most clients proxy via `bffCentraSearch` rather than hitting Centra's search endpoint from the browser.

**Account.** Build a `CentraAccountClient` that can be passed to `AccountProvider` as `accountClientDriver` when you mount one (wrap in `VoyadoEngageCentraAccountDriver` if Voyado is also mounted). Note: neither Stronger nor Dedicated mounts `AccountProvider` in its Centra bootstrap file — account state wires through `VoyadoEngageCentraAccountDriver` + `CentraAccountClient` at a different layer.

```tsx
import {
  CentraAccountClient,
  makeCentraCheckoutApiOrderToGreaseOrderMapper,
  centraCheckoutApiCustomerToGreaseCustomer,
} from "@made-people/grease-centra";

const centraAccountClient = new CentraAccountClient(
  centraCheckoutAPIClient,
  {
    orderMapper: makeCentraCheckoutApiOrderToGreaseOrderMapper(/* … */),
    customerMapper: centraCheckoutApiCustomerToGreaseCustomer,
  },
  { resetPasswordUri: "forgot-password", resetPasswordData: { id, i } },
);
```

## Session tokens

The Centra session token is sent on every Checkout API call. It also doubles as the "cart reference" for downstream integrations — Stronger passes `centraTokenHandler.getToken() ?? ""` into `VoyadoEngageProvider`, with a source-code note flagging the mapping as provisional.

## Gotchas

- **`"onInteraction"` is a no-op.** It logs an error and returns. Use `"onLoad"` or `"dontInitialize"`.
- **Checkout finalize re-fetches without a gate.** Pass `initializationType: "dontInitialize"` when `pageType === "checkoutFinalize"` or the finalized order gets overwritten.
- **`useCentraContext()` throws outside `CentraProvider`.**
- **`CentraCartClientDriver.updateComment` throws `"Not Implemented"`.** Don't wire a cart-comment UI to it.
- **`mapperOptions.options` is required.** Missing `productImageKey` or `context.currencyCode` breaks the product mapper at runtime.
- **`queryContext.site` attaches a `productUrl`.** When set, `addToCart` appends `{baseUrl}/{slug}/{product.slug}` to the Centra request — omit `site` to skip.

## Troubleshooting

**Cart empties on the checkout finalize page** — the provider re-fetched and replaced the finalized selection. Gate on `pageType === "checkoutFinalize"` with `"dontInitialize"`.

**`useCentraContext must be used within a CentraProvider`** — a consumer is mounted above `CentraProvider`. Check bootstrap order.

**Product search returns nothing** — `searchFunctionGetter` isn't wired, or the BFF search route isn't proxying to Centra.

## Related packages

- `@made-people/grease-cart` — auto-consumed via `CentraGreaseCartProvider` and `CentraCartClientDriver`.
- `@made-people/grease-order` — `makeCentraCheckoutApiOrderToGreaseOrderMapper` feeds the account client's `fetchOrders`.
- `@made-people/grease-account` — pair `CentraAccountClient` with `AccountProvider` for login, register, reset-password, and order history.
- `@made-people/centra-checkout-api` — the underlying HTTP client; `createCentraCheckoutAPIClient` and `CentraLocalStorageTokenHandler` live here, not in grease-centra.
