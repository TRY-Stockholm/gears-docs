## What this guide covers

How to mount `@made-people/grease-growthbook` in a React client frontend to read feature flags, bucket users into experiments, and push overrides into components wired with `withManagedProps` — without flicker on first paint.

## Prerequisites

- A GrowthBook account with an SDK Connection (client key + API host, e.g. `https://cdn.growthbook.io`).
- A server-rendered GrowthBook payload (`FeatureApiResponse`). Miss Mary and Stronger decode this from a `gbState` object delivered by the BFF.
- Every peer in `peerDependencies`, notably `@growthbook/growthbook`, `@growthbook/growthbook-react`, `@made-people/managed-props`, `@made-people/grease-ui-events`, `js-cookie`, `react`, `react-dom`, `zod`.

## Install

```bash
bun add @made-people/grease-growthbook
```

## Configure the client

Instantiate a `GrowthBook` client and hydrate it synchronously with the server payload — keeps flags stable between SSR and first client render.

```tsx
import { GrowthBook } from "@growthbook/growthbook";

const client = new GrowthBook({
  apiHost: "https://cdn.growthbook.io",
  clientKey: growthbookConfig.clientKey,
  enableDevMode: process.env.NODE_ENV !== "production",
  trackingCallback: (experiment, result) => {
    window.dataLayer = window.dataLayer ?? [];
    window.dataLayer.push({
      event: "growthbookExperimentViewed",
      growthbook: { experimentId: experiment.key, variationId: result.key },
    });
  },
}).initSync({
  payload: gbState.p,
  streaming: typeof window !== "undefined",
});
```

## Wire up the provider

`GreaseGrowthbookProvider` composes the GrowthBook React provider, an active-features store, a viewed-experiments context, and a `ManagedPropsProvider`. Mount it above anything that reads flags.

```tsx
import { GreaseGrowthbookProvider } from "@made-people/grease-growthbook";

<GreaseGrowthbookProvider
  clientProviderProps={{ client, viewedExperiments: gbState.e }}
  managedProps={{ fallbackProps: defaultProps }}
  deferredComponentNames={["HeroBlock"]}
>
  {children}
</GreaseGrowthbookProvider>;
```

If you omit `client`, pass `clientProviderProps.payload` instead — the provider throws `"GreaseGrowthbookClientProvider: You must provide either a client or payload"` otherwise.

## Use a feature flag (hook)

```tsx
import { useGreaseGrowthbookFeatureValue } from "@made-people/grease-growthbook";

function PaypalSlot() {
  const show = useGreaseGrowthbookFeatureValue("paypal-below-kco", false);
  return show ? <PaypalButton /> : null;
}
```

The hook wraps `useFeatureValue` and returns `undefined` if the provider is missing, so it is safe in shared components.

## Run an experiment (hook)

Wrap a variation with `GreaseGrowthbookExperimentContainer` — it reads viewed experiments from context and renders children only for the matching variation.

```tsx
import { GreaseGrowthbookExperimentContainer } from "@made-people/grease-growthbook";

<GreaseGrowthbookExperimentContainer experimentId="hero-copy" variationId="v2">
  <HeroV2 />
</GreaseGrowthbookExperimentContainer>;
```

Call `useGreaseGrowthbookViewedExperiments()` to read the full `experimentId -> variationId` map.

## User attributes

Three sources feed `GrowthBook.setAttributes`:

1. `clientProviderProps.attributes` — literal passthrough.
2. The browser cookie `gbuuid` (override via `clientProviderProps.cookieName`) — written to `attributes.id` on mount.
3. Route changes — the provider subscribes to `uiEventEmitter.on("routeChanged")` from `@made-people/grease-ui-events` and calls `client.setURL(url)` plus `client.setAttributes({ ...current, url, urlPathname })`.

Observe updates downstream via `greaseGrowthbookEventEmitter.on("attributesUpdated", ...)`.

## Gotchas

- `gbuuid` is only read browser-side, so SSR cannot target by `attributes.id`. Warm the cookie upstream or pass an explicit `attributes.id`.
- `deferredComponentNames` only defers components registered with `withManagedProps`; otherwise it is a no-op.
- Flags named `ComponentName::nested_prop_path` are exploded into managed-props overrides via `::` and `_` delimiters (both configurable).
- Hooks from `@growthbook/growthbook-react` throw outside the provider; prefer the `useGrease*` wrappers.
- The package does not export a `gbState` decoder or a default `trackingCallback`.

## Related packages

- `@growthbook/growthbook`, `@growthbook/growthbook-react` — underlying SDK.
- `@made-people/managed-props` — powers `withManagedProps` and receives resolved overrides.
- `@made-people/grease-ui-events` — supplies the `routeChanged` event that syncs URL attributes.
- `@made-people/grease-event-emitter` — emitter used for `attributesUpdated`.

## Troubleshooting

- **Provider throws on mount:** pass either `client` or `payload`.
- **Flags flicker on first paint:** add the component to `deferredComponentNames` and register it with `withManagedProps`, or hydrate via `initSync({ payload })`.
- **SSR and client disagree on a flag:** `gbuuid` is unset during SSR — warm it upstream or pass an explicit `attributes.id`.
- **`useFeatureValue` throws:** the component rendered outside the provider. Use `useGreaseGrowthbookFeatureValue` for a safe fallback.
