## What this guide covers

How to mount `@made-people/grease-gtm` in a React client frontend so Google Tag Manager is loaded and GA4-style ecommerce events (`add_to_cart`, `view_item`, `purchase`, etc.) are pushed automatically from grease cart/order/product emitters.

## Prerequisites

- A React client frontend that can import `@made-people/grease-gtm`
- One or more GTM container IDs (e.g. `GTM-XXXXXX`) — in the dedicated client these come from `rootLoaderData.settingsAndContentResponse.gtm?.containerIds`
- A currency code available at bootstrap (merged into every ecommerce push)
- Peer deps at the ranges in `peerDependencies`: `@made-people/grease-order`, `@made-people/grease-ui-events`, `@made-people/react-use-effect-once`, `react`, `sha.js`, `zod`

## Install

```bash
bun add @made-people/grease-gtm
```

## Load the GTM script

`GtmScriptHeadTag` injects the init snippet and `gtm.js` for every container; `GtmScriptNoscriptTag` renders the `<noscript>` iframe fallback. Both read container IDs from `GtmProvider` via `useGtmContainerIds` (or take a `containerIds` prop).

```tsx
import {
  GtmScriptHeadTag,
  GtmScriptNoscriptTag,
} from "@made-people/grease-gtm";

<head>
  <GtmScriptHeadTag />
</head>
<body>
  <GtmScriptNoscriptTag />
</body>
```

Optional `GtmScriptHeadTag` flags (both default `true`): `addOriginalLocation` (pushes `originalLocation` to prevent Google Ads rogue referrals) and `addFirstBrowserInteractionEvent` (a one-shot Pagespeed Insights trick — do not bind tracking to it).

## Wire up the provider

```tsx
import { GtmProvider, GtmProviderProps } from "@made-people/grease-gtm";
import { dedicatedGtmMappers } from "../mappers/dedicated-gtm-mappers";

const gtmProviderProps: GtmProviderProps = {
  containerIds:
    rootLoaderData.settingsAndContentResponse.gtm?.containerIds ?? [],
  globalFields: {
    ecommerce: { currency: frontendContext.currency.currency! },
  },
  mappers: dedicatedGtmMappers,
};

<GtmProvider {...gtmProviderProps}>{children}</GtmProvider>;
```

`mappers` is a `Partial<GtmMappers>` that can override `greaseProductToGtmItem`, `greaseLineItemToGtmItem`, `greaseAddressToGtmCustomer`, or `greaseProductBaseToLineItemBase`. Mounting the provider also mounts `GtmEventListeners`.

## Track events

You rarely call grease-gtm directly — the auto-mounted listeners push events when your existing grease APIs fire:

- `GtmEventAddToCart` / `GtmEventRemoveFromCart` — subscribe to `cartEventEmitter` and push `add_to_cart` / `remove_from_cart`
- `GtmEventViewItem` / `GtmEventViewItemList` / `GtmEventSelectItem` — subscribe to `productEventEmitter` and push `view_item`, `view_item_list`, `select_item`
- `GtmEventBeginCheckout`, `GtmEventAddShippingInfo`, `GtmEventAddPaymentInfo` — checkout funnel
- `GtmEventPurchase` — listens on `orderEventEmitter.orderPlaced`, pushes `purchase` with totals, vouchers, mapped line items
- `GtmEventLogin`, `GtmEventSignUp`, `GtmEventPageView`
- `GtmEventFeatureInteraction` — maps `uiEventEmitter.featureInteraction` to a custom `${feature}_${action}` event

For ad-hoc pushes:

```tsx
import { useGtm } from "@made-people/grease-gtm";

const { pushEcommerceEvent, pushCustomEvent } = useGtm();
pushCustomEvent({ event: "my_custom_event", foo: "bar" });
```

`pushEcommerceEvent` is typed against `GtmEcommerceEvent` (a zod discriminated union on `event`); invalid payloads are `console.error`-logged and dropped.

## Consent mode

Not supported out of the box — there is no `consent` code in the package. Either inject `gtag('consent', ...)` in your own head markup, or gate pushes with `registerDataLayerBeforeHook`:

```ts
import { registerDataLayerBeforeHook } from "@made-people/grease-gtm";

registerDataLayerBeforeHook((data) => {
  if (!userHasConsented()) return false;
});
```

## Gotchas

- `useGtm` throws outside a `GtmProvider`; use `useGtmContainerIds` for script tags.
- Bad ecommerce payloads fail zod and are dropped silently (error logged).
- Each ecommerce push first writes `{ ecommerce: null }` to reset GTM's auto-merging.
- `gtmDataLayerPushEvent` warns and noops when `window` is undefined (SSR-safe).
- `firstBrowserInteraction` is a perf hint, not an analytics event.
- Wiring verified in `client-frontends/dedicated/app/project/bootstrap/bootstrap.tsx` and `client-frontends/dedicated/app/root.tsx`. Other clients not checked.

## Troubleshooting

- **No events in GTM preview:** `containerIds` was empty at mount — `GtmScriptHeadTag` returns `null` and never injects.
- **Ecommerce event missing:** zod rejected it; check the console error from `gtmDataLayerPushEcommerceEvent`.
- **Duplicate items on quantity change:** `GtmEventAddToCart` only pushes a diff when the new quantity is higher.
- **Purchase missing items:** ensure any `greaseLineItemToGtmItem` override still spreads `defaultGtmMappers.greaseLineItemToGtmItem(...)`.
