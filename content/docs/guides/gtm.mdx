## What you'll learn

How to mount `@made-people/grease-gtm` so GTM loads and GA4-style ecommerce events push automatically from grease cart, order, and product emitters.

## The four things to know

1. **Two script tags, one provider.** `GtmScriptHeadTag` injects `gtm.js` client-side (inside a `useEffectOnce`; returns `null` on SSR). `GtmScriptNoscriptTag` renders the `<noscript>` iframe. `GtmProvider` supplies container IDs, global fields, and mappers.
2. **Events are emitter-driven.** You almost never call grease-gtm directly. Listeners inside the provider subscribe to grease emitters and push GA4 events for you.
3. **Payloads are zod-validated.** Ecommerce pushes run through `gtmEcommerceEventSchema`. Invalid payloads log an error and drop silently.
4. **Consent is not included.** The package ships no `gtag('consent', ...)` call. Wire it outside, or gate pushes with `registerDataLayerBeforeHook`.

## 1. Install

Package: `@made-people/grease-gtm` (0.19.5). Peer deps: `@made-people/grease-order ~0.10.11`, `@made-people/grease-ui-events ~0.4.1`, `@made-people/react-use-effect-once ~0.3.1`, `react 18.2.0`, `sha.js ~2.4.11`, `zod ^3.22.4`.

```bash
# Inside the monorepo
rush add -p @made-people/grease-gtm

# Outside the monorepo
pnpm add @made-people/grease-gtm
```

## 2. Load the scripts

```tsx
import {
  GtmScriptHeadTag,
  GtmScriptNoscriptTag,
} from "@made-people/grease-gtm";

<head>
  <GtmScriptHeadTag />
</head>
<body>
  <GtmScriptNoscriptTag />
  {/* … */}
</body>
```

Both tags read container IDs from `GtmProvider` via `useGtmContainerIds`, or accept a `containerIds` prop. `GtmScriptHeadTag` has two optional flags, both default `true`: `addOriginalLocation` (prevents Google Ads rogue referrals) and `addFirstBrowserInteractionEvent` (a Pagespeed Insights perf trick — not a tracking signal).

> **Client-side only.** All script injection happens client-side inside a `useEffectOnce`. `GtmScriptHeadTag` returns `null` on SSR and also returns `null` if `containerIds` is empty at mount — no retry.

## 3. Mount the provider

```tsx
import { GtmProvider, GtmProviderProps } from "@made-people/grease-gtm";
import { dedicatedGtmMappers } from "../mappers/dedicated-gtm-mappers";

const gtmProviderProps: GtmProviderProps = {
  containerIds:
    rootLoaderData.settingsAndContentResponse.gtm?.containerIds ?? [],
  globalFields: { ecommerce: { currency: frontendContext.currency.currency! } },
  mappers: dedicatedGtmMappers,
};

<GtmProvider {...gtmProviderProps}>{children}</GtmProvider>;
```

`mappers` is a `Partial<GtmMappers>`. Override any of `greaseProductToGtmItem`, `greaseLineItemToGtmItem`, `greaseAddressToGtmCustomer`, or `greaseProductBaseToLineItemBase`. Mounting `GtmProvider` also mounts `GtmEventListeners`.

`globalFields` is typed `{ ecommerce: { currency: string } } & Record<string, unknown>` — the `ecommerce.currency` shape is part of the contract, not arbitrary. Add any other top-level fields you want merged into every push alongside it.

Need to tweak what the listeners do? `GtmProvider` also takes an optional `eventListeners?: GtmEventListenersProps` escape hatch — e.g. a `selectItem` sub-config — for turning listeners off or customizing their behavior.

## 4. What the listeners push

| Listener | Emitter | GA4 event |
|---|---|---|
| `GtmEventAddToCart` / `RemoveFromCart` | `cartEventEmitter` | `add_to_cart`, `remove_from_cart` |
| `GtmEventViewItem` / `ViewItemList` / `SelectItem` | `productEventEmitter` | `view_item`, `view_item_list`, `select_item` |
| `GtmEventBeginCheckout` / `AddShippingInfo` / `AddPaymentInfo` | checkout funnel | funnel events |
| `GtmEventPurchase` | `orderEventEmitter.orderPlaced` | `purchase` with totals, vouchers, items |
| `GtmEventLogin` / `SignUp` / `PageView` | auth + nav | `login`, `sign_up`, `page_view` |
| `GtmEventFeatureInteraction` | `uiEventEmitter.featureInteraction` | custom `${feature}_${action}` |

## 5. Ad-hoc pushes

```tsx
import { useGtm } from "@made-people/grease-gtm";

const { pushCustomEvent, pushEcommerceEvent } = useGtm();
pushCustomEvent({ event: "my_custom_event", foo: "bar" });
```

`pushEcommerceEvent` validates against `GtmEcommerceEvent` (a zod discriminated union on `event`). Invalid payloads are logged via `console.error` and dropped.

## 6. Consent mode

Either inject `gtag('consent', ...)` in your own head markup, or gate pushes:

```ts
import { registerDataLayerBeforeHook } from "@made-people/grease-gtm";

registerDataLayerBeforeHook((data) => {
  if (!userHasConsented()) return false;
});
```

Returning `false` aborts the push before it reaches `dataLayer`. For tests, `resetDataLayerBeforeHooks()` is exported alongside `registerDataLayerBeforeHook` to clear registered hooks between cases.

## Render tree

```
<head>
  └─ <GtmScriptHeadTag />          ← injects gtm.js per container
<body>
  ├─ <GtmScriptNoscriptTag />      ← first child, iframe fallback
  └─ <GtmProvider>                 ← IDs, globalFields, mappers
      └─ <GtmEventListeners />     ← auto-mounted; subscribes to emitters
          └─ app
```

## Gotchas

- `useGtm` throws outside a `GtmProvider`. Use `useGtmContainerIds` in the script tags — it returns `[]` when no provider.
- Bad ecommerce payloads fail zod and drop silently; check the console.
- Each ecommerce push first writes `{ ecommerce: null }` to reset GTM's auto-merging.
- `gtmDataLayerPushEvent` warns and noops when `window` is undefined — SSR-safe.
- `firstBrowserInteraction` is a perf hint, not an analytics event. Do not bind GA or FB tracking to it.

## Troubleshooting

**No events in GTM preview** — `containerIds` was empty at mount, so `GtmScriptHeadTag` returned `null`. Check `rootLoaderData.settingsAndContentResponse.gtm?.containerIds`.

**Ecommerce event missing** — zod rejected it. Look for the `console.error` from `gtmDataLayerPushEcommerceEvent`.

**Quantity decrease not tracked** — `GtmEventAddToCart` only pushes when quantity increases (`qtyDiff > 0`); decreases via `updateQuantity` are silently dropped. `remove_from_cart` fires only on real `removeFromCart` events, not on `updateQuantity` decrements.

**Purchase missing items** — a `greaseLineItemToGtmItem` override isn't spreading `defaultGtmMappers.greaseLineItemToGtmItem(...)`. Merge, don't replace.
