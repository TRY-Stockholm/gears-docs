## What you'll learn

How to mount `@made-people/grease-voyado-engage` so a client can identify contacts, subscribe to newsletters, surface promotions, and emit tracking events.

## The four things to know

1. **One provider does it all.** `VoyadoEngageProvider` holds the API client, discovery key, frontend context, and tracking toggle. Everything else reads from it.
2. **It goes inside Centra.** The provider needs a `cartReference` from Centra's token handler, so it must sit below `CentraBootstrap` in the stack.
3. **Drivers plug into sibling greases.** `VoyadoEngageNewsletterClient`, `VoyadoEngageBackInStockClient`, and `VoyadoEngageCentraAccountDriver` are the adapters that hand Voyado to `grease-newsletter`, `grease-back-in-stock`, and `grease-account`.
4. **Tracking depends on `window.va`.** The Voyado tracking script must be loaded separately. If `window.va` is missing, a call waits 1 second then attempts once; if `window.va` is still undefined, it logs a `console.warn` and drops the event.

## 1. Create the API client

```tsx
import { createVoyadoEngageClient } from "@made-people/voyado-engage-api";

const voyadoEngageClient = createVoyadoEngageClient({
  baseUrl: useGearsServiceUrl("engage"),
});
```

> **Never hard-code the base URL.** Resolve it from gears service config so preview, local, and prod all point at the right tenant.

## 2. Mount the provider

```tsx
import { VoyadoEngageProvider } from "@made-people/grease-voyado-engage";

<VoyadoEngageProvider
  apiClient={voyadoEngageClient}
  frontendContext={{
    marketId: frontendContext.market,
    countryCode: frontendContext.country,
    marketName: frontendContext.marketName ?? "SE",
    language: voyadoEngageLanguageMapper(frontendContext.country),
    baseUrl: frontendContext.baseUrl,
    siteSlug: frontendContext.siteSlug,
    cartReference: getSetVoyadoCartReference(),
    engageLocale: voyadoEngageLocaleMapper(frontendContext.country ?? "se"),
  }}
  discoveryKey={getSetVoyadoCustomerKey()}
  setDiscoveryKey={setVoyadoCustomerKey}
  withTracking
  softIdentification={{ encryptedContactId: eclub }}
>
  {children}
</VoyadoEngageProvider>;
```

Where each field comes from:

| Field | Source |
|---|---|
| `marketId`, `countryCode`, `marketName`, `baseUrl`, `siteSlug` | `FrontendContextV1` (BFF) |
| `language`, `engageLocale` | Client-specific mappers (not exported) |
| `cartReference` | A stable cart reference string the client manages — Dedicated uses a client-local `getSetVoyadoCartReference()` helper (see `client-frontends/dedicated/app/project/bootstrap/engage/utils/get-set-voyado-cookies.ts`) |
| `discoveryKey` | localStorage helper (e.g. `getSetVoyadoCustomerKey`) |
| `encryptedContactId` | URL param `?eclub=...` |

A real example lives at `client-frontends/dedicated/app/project/bootstrap/engage/engage-bootstrap.tsx`.

## 3. Newsletter

```tsx
import { VoyadoEngageNewsletterClient } from "@made-people/grease-voyado-engage";

const newsletterClientDriver = new VoyadoEngageNewsletterClient(
  voyadoEngageClient,
  {
    market: frontendContext.marketName ?? "SE",
    countryCode: frontendContext.country,
    contactType: "contact",
    withTracking: true,
    preferences: { acceptsEmail: true, acceptsSms: true },
    attributes: { lang: voyadoEngageLanguageMapper(frontendContext.country) },
    discoveryKey: getSetVoyadoCustomerKey(),
  },
  { setDiscoveryKey: setVoyadoCustomerKey },
);
```

Pass `newsletterClientDriver` to `NewsletterProvider` from `@made-people/grease-newsletter`.

> **`contactType: "contact"` vs `"member"`.** The Dedicated newsletter driver uses `"contact"`. `"member"` is only used for account-creation signup via `VoyadoEngageCentraAccountDriver.subscribeOptions` — don't use it in a newsletter-only context.

## 4. Promotions

```tsx
import { useVoyadoEngageContactPromotions } from "@made-people/grease-voyado-engage";

const {
  promotions,
  addEngagePromotionCodeToCart,
  removeEngagePromotionCodeFromCart,
} = useVoyadoEngageContactPromotions();
```

Auto-fetches ECOM-redeemable promotions for the logged-in contact. `voucherCode` is what the user enters; `promotionId` is Voyado's internal id. Both are required to apply.

## 5. Tracking

```tsx
import { engageTrackProductView } from "@made-people/grease-voyado-engage";

engageTrackProductView({ productId, category, locale });
```

The product must expose a `voyadoEngageTracking` attribute of shape `{ productId: string; category?: string | null }` — typically wired via a Centra attribute. Pull `productId` and `category` off that attribute, and pass the current `locale` when calling `engageTrackProductView`.

## Gotchas

- **`window.va` must be loaded separately.** Voyado's tracking script is not bundled. Without it, a tracking call waits 1 second then attempts once; if `window.va` is still undefined, it logs a `console.warn` and drops the event.
- **Products without `voyadoEngageTracking` log a warning and do nothing.** Check Centra attributes.
- **`voyadoEngageLanguageMapper` / `voyadoEngageLocaleMapper` are not exported.** Every client writes its own country-to-locale mapping.
- **`discoveryKey` is session-scoped.** Stored in localStorage. Clearing storage resets it.
- **`?eclub=<id>` only fires if `softIdentification` is passed.** The URL param alone is ignored.
- **`VoyadoEngageNewsletterClient.unsubscribe()` is not implemented.** It throws.
- **Attribute fetching on login is opt-in.** `VoyadoEngageCentraAccountDriver` only calls Voyado for custom attributes when `enableAttributeFetching: true`.

## Troubleshooting

**Tracking calls do nothing** — `window.va` is not defined. Confirm the Voyado tracking script is loaded before any event fires.

**Product view not tracked** — the product is missing the `voyadoEngageTracking` attribute. Check the product data source.

**Soft identification does nothing** — you didn't pass `softIdentification` to the provider. The `?eclub=` param is only read when that prop exists.

**"useVoyadoEngage must be used within a Provider"** — a hook is mounted above `VoyadoEngageProvider`. Check provider order in `bootstrap.tsx`.
