## What you'll learn

How Gears emits product data to external systems through two HTTP feeds — the **Algolia admin product feed** (consumed by Algolia's indexer) and the **Google Shopping product feed** (consumed by Google Merchant Center) — and how to reach each one for every Gears brand in test and production.

Sitemaps and `robots.txt` are a related kind of raw-response route but live in their own guide; see [SEO Routes](./seo-routes).

## The two feeds at a glance

| Feed | Service | Endpoint | Consumer |
|---|---|---|---|
| Algolia product feed | `algolia` | `GET /admin/feeds/products/:siteSlug` | Algolia indexer (admin-gated push) |
| Google Shopping | `bff` | `GET /v1/google-shopping/:siteSlug` | Google Merchant Center (pulls XML) |

Both endpoints are scoped by `:siteSlug` — the per-market slug (e.g. `stronger-se`, `miss-mary-no`) configured under `sites` in the tenant config. One brand typically exposes several site slugs; pick the right one for the market you're feeding.

## 1. Algolia product feed

Generated by `AlgoliaFeedsController` in `nestjs-algolia-service`. The controller surface is shown in [Swagger / OpenAPI](./swagger):

```ts
@ApiTags("Algolia Feeds")
@Controller(ADMIN_FEED_CONTROLLER_PREFIX)
export class AlgoliaFeedsController {
  @Get("/products/:siteSlug")
  @ApiOperation({ summary: "Generate the Algolia product feed for a site" })
  @ApiResponse({ status: 200, type: ProductFeedResponseV1 })
  public async generate(
    @Param("siteSlug") siteSlug: string,
  ): Promise<ProductFeedResponseV1> {
    return this.feedService.generate(siteSlug);
  }
}
```

Key facts:

- **Admin-gated.** The controller sits behind `ApiKeyAuthMiddleware` — set `x-admin-api-key` to the value of `admin.apiKey` in tenant config when calling from outside the cluster. The Swagger UI's "Try it out" button does **not** auto-attach the header; paste the key into the `x-admin-api-key` field at the top of the page.
- **Returns JSON.** `ProductFeedResponseV1` is a structured shape, not raw XML — different from the Google feed.
- **One call per site slug.** Brands with multiple markets index each market separately; loop the slugs upstream.

### URL formula

```
https://<tenant>.algolia.<env>.gears.madepeople.dev/admin/feeds/products/<siteSlug>
```

`<env>` is `test` or `prod`. `<tenant>` is the kebab-less brand id (`astridlindgren`, not `astrid-lindgren`).

## 2. Google Shopping feed

Generated by the BFF and exposed as part of its `Google` route group ([The BFF](./bff#google)):

| Route | Method | Purpose |
|---|---|---|
| `v1/google-shopping/:siteSlug` | GET | Google Shopping product feed |
| `v1/google-shopping/:siteSlug/list-invalid-products` | GET | Debug view of feed exclusions |

Key facts:

- **Public read.** Google Merchant Center pulls the URL on a schedule; the endpoint is not admin-gated. Whatever the BFF emits, Google sees.
- **Driven by `tracking.feeds.google` in tenant config.** Per-market overrides for currency, country, base URL, and excluded categories live there.
- **Exclusions are explicit.** Products are dropped from the feed for reasons like `globalNoIndex`, out-of-stock, missing required Merchant Center fields, or a market-specific exclusion list. Hit `/list-invalid-products` for the same site slug to see exactly which products were filtered and why — invaluable when Merchant Center reports a count mismatch.

### URL formulas

```
Feed       : https://<tenant>.bff.<env>.gears.madepeople.dev/v1/google-shopping/<siteSlug>
Debug list : https://<tenant>.bff.<env>.gears.madepeople.dev/v1/google-shopping/<siteSlug>/list-invalid-products
```

## 3. Sitemaps and robots.txt

The sitemap index, per-market product / story sitemaps, and `robots.txt` are also outbound, but they're served as raw-response routes from each client's Remix layer — see [SEO Routes](./seo-routes) for the route → loader → BFF controller pipeline and the Miss Mary per-market domain rewrite pattern.

## Gotchas

- **The Algolia feed UI doesn't auto-attach `x-admin-api-key`.** "Try it out" returns 401 until you paste the key into the header field at the top of the Swagger page. The CLI equivalent: `curl -H "x-admin-api-key: $KEY" https://<tenant>.algolia.prod.gears.madepeople.dev/admin/feeds/products/<siteSlug>`.
- **`siteSlug` must match a configured site.** Passing `miss-mary` when the configured slugs are `miss-mary-se`, `miss-mary-no`, etc. yields an empty/404 response, not a friendly error. Cross-check against `sites` in tenant config.
- **Google Merchant Center caches aggressively.** A change to `tracking.feeds.google` won't show up in Merchant Center until the next scheduled pull (typically nightly). For an immediate read, hit the URL in a browser — that's the same response Google will see.
- **`list-invalid-products` is the debug surface, not a feed.** Merchant Center should not be pointed at it. Use it only when investigating count drift.
- **Yotpo and other tenant-scoped services aren't deployed for every brand.** The tables above cover every brand for `bff` and `algolia` specifically — those two are deployed for all six brands in both envs.

## Troubleshooting

**Algolia feed returns 401** — `x-admin-api-key` is missing or wrong. Paste the value of `admin.apiKey` from tenant config into the header field at the top of the Swagger UI, or attach it via `curl -H "x-admin-api-key: $KEY"`.

**Algolia feed returns an empty product list** — The `siteSlug` doesn't match a configured site, or the Centra cache for that site is empty. Run `PUT /v1/admin/cache-collector/centra/products` on the BFF first, then re-hit the feed.

**Google Shopping feed and Merchant Center disagree on product count** — Hit `/v1/google-shopping/<siteSlug>/list-invalid-products`. The response lists every excluded product and the reason (`out-of-stock`, `globalNoIndex`, `missing-gtin`, etc.). Reconcile from there.

**Feed shows the wrong domain on product URLs** — The BFF emits canonical URLs from tenant config. If the brand serves a different domain per market (Miss Mary's `.se` / `.no` / `.dk`), the rewrite pattern in [SEO Routes §7](./seo-routes#7-per-client-response-transforms) shows how to fix it client-side for sitemaps; the same shape applies if a feed needs per-market rewriting.

**404 on the Swagger UI itself** — That brand's service isn't deployed in that environment. Algolia and BFF are deployed everywhere, but if you wander into Yotpo, Recrql, Voyado Engage or Config, availability varies per tenant — confirm against the cluster's service map.
